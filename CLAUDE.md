## Architecture

### Data flow

All public functions follow the same pattern:

1.  Validate inputs (`check_id()`, `check_date()`, `check_bbox()` in
    `R/utils.R`)
2.  Call `dt_get_json()` or `dt_get_csv()` (`R/client.R`)
3.  Pass the response to a `parse_*()` function (`R/parsers.R`)
4.  Return a tidy tibble

### HTTP client (`R/client.R`)

`dt_base_request()` builds an httr2 request with a shared User-Agent,
30-second timeout, retry logic (up to 5× on 429/500/503), and a 10
req/min throttle. `dt_perform()` wraps `req_perform()` and converts HTTP
errors into classed `cli_abort()` conditions
(`digitraffic_error_not_found`, `digitraffic_error_rate_limit`,
`digitraffic_error_network`, `digitraffic_error_http`). All functions
call either `dt_get_json()` or `dt_get_csv()`, which compose a request
and return parsed body.

### Caching (`R/cache.R` and `R/dt_stations_detailed.R`)

Two independent caching layers:

- **Session cache** (`.dt_cache` environment):
  `dt_cache_set()`/`dt_cache_get()` with a 5-minute TTL. Used by
  [`dt_stations()`](https://bbtheo.github.io/digitraffic/reference/dt_stations.md)
  and
  [`dt_sensors()`](https://bbtheo.github.io/digitraffic/reference/dt_sensors.md)
  to avoid redundant API calls within a session.
- **Detailed station cache**: Stores municipality/province/road metadata
  for all ~490 stations. Priority order: (1) user disk cache at
  `tools::R_user_dir("digitraffic", "cache")/stations_detailed.rds`, (2)
  bundled snapshot in `R/sysdata.rda` (the object
  `stations_detailed_cache`, regenerated by
  `data-raw/stations_detailed.R`).
  [`dt_stations_load_details()`](https://bbtheo.github.io/digitraffic/reference/dt_stations_load_details.md)
  fetches all station details in parallel via
  [`httr2::req_perform_parallel()`](https://httr2.r-lib.org/reference/req_perform_parallel.html)
  and saves to the disk cache.

### Station identifiers

Two distinct IDs exist per station — they are **not** interchangeable:

- `id`: internal API identifier, used by
  [`dt_station()`](https://bbtheo.github.io/digitraffic/reference/dt_station.md)
  and
  [`dt_station_data()`](https://bbtheo.github.io/digitraffic/reference/dt_station_data.md)
  (JSON endpoints)
- `tms_number`: TMS system number, used by
  [`dt_history_raw()`](https://bbtheo.github.io/digitraffic/reference/dt_history_raw.md)
  (CSV endpoint URL path)

### Parsers (`R/parsers.R`)

All JSON → tibble and CSV → tibble conversions are internal. GeoJSON
FeatureCollections are flattened by `parse_stations_response()`. The
`%||%` null-coalescing operator (`R/utils.R`) is used throughout to
handle missing API fields gracefully. Historical CSV files have no
header — column names and types are defined in `.lam_csv_col_names` /
`.lam_csv_col_types` and parsed with
[`readr::read_delim()`](https://readr.tidyverse.org/reference/read_delim.html).

### Testing

Tests use `httptest2::with_mock_dir("fixtures", {...})` to replay
recorded API responses stored under
`tests/testthat/fixtures/tie.digitraffic.fi/`. Tests that need to assert
on specific HTTP error classes use
[`httr2::with_mocked_responses()`](https://httr2.r-lib.org/reference/with_mocked_responses.html)
directly. The `helper.R` file loads httptest2 for all test files.
